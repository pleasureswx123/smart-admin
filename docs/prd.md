# 【灵办】智能行政服务中心 - 产品需求文档 (PRD) v2.0
**（升级版：基于 LangChain RAG 与 LangGraph 多步推理智能体）**

## 1. 业务目标与核心场景
**核心目标：** 打造具备“反思与修正”能力的数字行政大脑。不仅能回答问题，还能独立完成复杂的行政策划与多流程访客接待。
**核心场景：**
1. **智能问答：** 基于 RAG 架构的深度制度咨询（支持跨文档关联）。
2. **反思写作：** 生成行政公文并自动检查合规性与语气。
3. **循环策划：** 根据预算自动搜索、筛选、并循环修正团建方案。
4. **访客管家：** 自动化登记并智能匹配内部接待人员。

## 2. 功能模块定义 (Feature List)

### 2.1 智能 RAG 知识助手 (Agentic RAG)
*   **多文档路由：** AI 能根据问题自动判断去哪个文档（如：请假去《考勤手册》，报销去《财务制度》）检索。
*   **检索回审：** AI 检索到内容后，先自评“内容是否能回答用户问题”，若不能，则扩大检索范围或追问员工。
*   **溯源显示：** 必须在回答中高亮引用文档的切片来源。

### 2.2 闭环公文 Copilot (Reflective Writer)
*   **写审分离：**
    *   **Node A (写作):** 根据需求生成公文初稿。
    *   **Node B (审计):** 检查初稿是否有错别字、是否违反合规性、语气是否达标。如果不合格，发回 Node A 重写。
*   **一键排版：** 输出标准的 Markdown，支持一键转 PDF。

### 2.3 团建策划智能体 (LangGraph Planner) - **重点功能**
*   **多步规划流程：**
    1. **搜索节点：** 根据关键词搜索真实餐厅/场地。
    2. **比对节点：** 汇总价格、距离、评价。
    3. **预算检查节点：** 若总额超标或距离太远，自动触发“重找”逻辑，重新搜索。
    4. **方案生成：** 输出最终带地图链接和行程表的 PDF。

### 2.4 智能访客收发室
*   **多模态登记：** 拍照名片/身份证，AI OCR 提取。
*   **模糊员工匹配：** 若访客填写的被访人是绰号或缩写，AI 通过通讯录语义匹配最可能的员工。
*   **自动推送：** 联动钉钉 API，发送到访卡片。

## 3. 核心业务流程 (User Flow)
*   **策划流程：** `输入预算/人数` -> `AI Agent 联网搜索` -> `自检预算是否超标` -> `(不合格重搜)` -> `合格后输出方案`。
*   **问答流程：** `提问` -> `Query 改写` -> `向量检索` -> `内容评分` -> `生成回答`。

## 4. LangGraph 逻辑节点规划 (给 Cursor 的逻辑备忘)
1. **Event_State:** 定义包含 `requirements`, `search_results`, `budget_ok`, `final_plan` 的状态机。
2. **Knowledge_Chain:** 使用 LangChain 的 `ConversationalRetrievalChain` 处理带上下文的问答。
3. **Audit_Loop:** 在公文写作中引入 `Conditional_Edges`（条件边），根据审计结果决定结束或重写。

## 5. 待解决的关键逻辑
1. **RAG 精度：** 如何解决行政文档中表格的识别问题（建议利用 AI 的视觉能力或 Markdown 解析）。
2. **联网实时性：** Tavily/Google Search 的数据过滤逻辑，排除已倒闭或评分过低的商户。
3. **权限：** 制度查询需区分部门权限（如：薪资福利仅限管理层）。
