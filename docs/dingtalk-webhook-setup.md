# 钉钉自定义机器人配置指南

本文说明如何获取 `DINGTALK_WEBHOOK_URL` 和 `DINGTALK_SECRET` 两个配置值。

---

## 前置条件

- 拥有一个钉钉账号，且已加入目标群聊（推荐新建一个专用的"访客通知"群）
- 使用**电脑端**钉钉完成以下操作（手机端不支持添加机器人）

---

## 操作步骤

### 第一步：打开群设置

1. 在钉钉电脑端，打开你要接收通知的群聊
2. 点击右上角 **「群设置」** 图标（齿轮 ⚙）
3. 在群设置面板中，找到并点击 **「智能群助手」**

### 第二步：添加自定义机器人

1. 点击 **「添加机器人」**
2. 在机器人列表中选择 **「自定义」**（Custom Robot）
3. 点击 **「添加」**

### 第三步：配置机器人

| 字段 | 填写内容 |
|------|---------|
| 机器人名字 | 访客通知助手（可自定义） |
| 安全设置 | 选择 **「加签」**（推荐，系统使用此方式） |

> ⚠️ **安全设置必须选「加签」**，不要选「关键词」或「IP 地址」，否则签名校验会失败。

### 第四步：获取两个配置值

勾选同意服务条款后点击「完成」，页面会显示：

```
Webhook 地址：https://oapi.dingtalk.com/robot/send?access_token=xxxxxxxx
加签密钥：   SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- **`DINGTALK_WEBHOOK_URL`** = 上方完整的 Webhook 地址（含 `access_token`）
- **`DINGTALK_SECRET`** = 加签密钥（以 `SEC` 开头的字符串）

### 第五步：填入 .env 文件

```env
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=你的token
DINGTALK_SECRET=SEC你的密钥
```

填写完毕后**重启后端**即可生效：

```powershell
# 在 backend/ 目录下
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

---

## 验证是否生效

在访客管理页面登记一条测试访客，若填写了被访人，系统会在登记成功后自动向群聊发送钉钉通知。  
访客列表中该条记录的「推送状态」会从 **待推送** 变为 **已推送**。

---

## 常见问题

| 现象 | 原因 | 解决方式 |
|------|------|---------|
| 推送状态一直显示「推送失败」 | Webhook URL 或 SECRET 填写有误 | 检查是否有多余空格或换行 |
| 收不到消息但状态是「已推送」 | 机器人被踢出群 / 群解散 | 重新添加机器人，更新 Webhook URL |
| `sign` 校验失败（后端日志报 310000） | 安全设置选了「关键词」而非「加签」 | 重新创建机器人，选择「加签」 |
| DINGTALK_SECRET 留空 | 不使用签名校验（不推荐） | 安全设置选「自定义关键词」时可留空，但存在安全风险 |
